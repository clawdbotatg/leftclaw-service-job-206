Audit Findings Fixed

HIGH 1Issuer Revocation Bypass — FIXED
Three independent flags: revokedByOwner, revokedByIssuer, revokedByAdmin. Each actor can only clear their own flag. No actor can override another's revocation.
HIGH 2Admin Revocation Bypass via Recovery — FIXED
recoverVault() and setRecoveryWallet() both check isRevoked() and revert if any revocation flag is active. Recovery cannot launder a revoked credential.
HIGH 3Deactivated Issuer Self-Reactivation — FIXED
registerOrUpdateIssuerProfile() only sets active = true on first registration. Updates never touch the active flag. Only owner can reactivate via reactivateIssuer().
MED 1Instant Registry Swap — FIXED
2-step update: proposeRegistry() + applyRegistryProposal() with 48-hour delay. Initial setup bypasses delay only when registry == address(0). Cancellation supported.
MED 2Lapsed Issuer Admin Rights — IMPLEMENTED
Lapsed subscription ≠ deactivated. issuerAdminRights(issuer) returns true if profile is active, regardless of subscription. Only admin deactivation removes rights. New minting requires active subscription.
LOW 1extendExpiry Reactivates Revoked Membership — FIXED
membership.active is always derived in the getter. Never stored as a static boolean. Automatically false if credential is revoked, burned, or membershipExpiry has passed.
LOW 2USDC Blocklist DoS — ADDRESSED
setFeeRecipient() is owner-updateable. SafeERC20 used throughout. Documented in contract comments.
LOW 3Unchecked Mint Counter Wrap — FIXED
setBasicMintQuota() requires quota < type(uint32).max. Constructor validates this. Counter explicitly checked before increment.
INFO 2vaultContract Unset at Deployment — FIXED
Deployment script links registry and vault atomically before transferring ownership. Both registry.vaultContract and vaultId.registry are set in the same deployment transaction.
